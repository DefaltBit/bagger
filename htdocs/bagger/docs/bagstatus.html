Bagger User Guide: Checking Bag Status

Checking that a bag is complete

    Basic flow:
  • 1. User selects to check that a bag is complete.
  • 2. Application checks that a bag is complete.
  • 3. If a BagInfo profile has been selected, application checks that it has been satisfied.
  • 4. Application updates the displayed information with the result, including warning about any unsatisfied fields from BagInfo profile.
    Note:
  • * A user may want to check that a bag is complete instead of verifying a bag because it is less time-consuming.

Checking that a bag is valid

    Basic flow:
  • 1. User selects to check that a bag is valid.
  • 2. Application checks that a bag is valid.
  • 3. If a BagInfo profile has been selected, application checks that it has been satisfied.
  • 4. Application updates the displayed information with the result, including warning about any unsatisfied fields from BagInfo profile.

The bag information has been updated and now all the bag tests pass. The bag has now been validated successfully. The bag is complete, the bag is valid and the bag required fields are valid, and the bag has been successfully saved over the previously existing bag.
