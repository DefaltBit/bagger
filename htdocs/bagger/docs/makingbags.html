Bagger User Guide: Making Bags

Creating a new bag

To create a new bag from scratch the user first selects the ‘Create New Bag’ button from the Bag Content Control buttons at the top of the application. This creates an empty bag with blank values for the Details input form and resets and previously selected buttons and options to their default values. The user does not yet need to enter a bag name since the bag does not yet exist on disk. This name is set when the user is ready to save the bag.

    Basic flow:
  • 1. User selects to create a new bag.
  • 2. User selects a bag version.
  • 3. Application clears any displayed information from previous bag and creates a new bag.

Notice that now the new bag has enabled the ‘Add Data’ button. The user can now begin to add data files to the bag.

Open an existing bag on disk

The Bagger Application can also load an existing bag folder or serialized file. This could be done to check whether the folder contains a valid bag or to add more data or more information to an existing bag. Start by selecting the ‘Open Existing Bag’ button from the Bag Content Control buttons at the top of the application. This displays a directory and file chooser dialog window. Select the desired bag from the file chooser dialog and then select the Open button. This will load the directory or file and create a bag if one exists.

    Basic flow:
  • 1. User selects to open an existing bag.
  • 2. User selects a bag on disk. (User provides file/directory using a File/Directory picker. User should have option to limit display of files by serialization format.)
  • 3. Application creates the bag.
  • 4. Application clears any displayed information from the previous bag and displays information from new bag.

Note: You do not need to close the current bag you are working on. It will start a new bag in its place, but if you have not already saved the current bag it will be overwritten with the new bag.

The existing bag has been loaded and its data contents are now displayed in the data hierarchy file tree. Notice that all the buttons are now enabled: the Add Data, Save Bag, Save Bag As, Validate Bag and Update Bag buttons are all enabled. You may now validate this bag, or add additional data, or edit the Details information and update it. Also notice that the Serialized Bag Type is back to being ‘none’.

Note: Since the existing bag has no concept of Profiles, the Send to org and Send to contact information for the loaded Bag will overwrite whatever was loaded for the currently selected project, however the Send from contact is not changed since this is not part of the bag.

Creating a bag-in-place

    Basic flow:
  • 1. User selects to create a bag-in-place.
  • 2. User selects a directory on disk. (User provides file/directory using a File/Directory picker. User should have option to limit display of files by serialization format.)
  • 3. User selects a version and whether to retain the base directory.
  • 4. Application creates the bag, including moving the directory/files as necessary to create the bag structure, completing the bag, and writes to disk.
  • 5. Application clears any displayed information from the previous bag and displays information from new bag.

    Notes:
  • * The directory that is bagged in place may be:
    • o A set of files/directories in the basic structure of bag a bag (i.e., it has a data directory with content), but is not complete.
    • o A directory containing a set of files/directories.
