The Bagger application was created for the U.S. Library of Congress as a tool to produce a package of data files according to the BIL (BagIt Library) specification (see BIL Overview section). It can create a structured directory of files and folders, or it can create a serialized compressed archive file of the directory, such as a zip or tar file. For this document, 'bag' refers to BIL's representation of a bag. This bag is in-memory and may refer to content on disk. 'Bag on disk' refers to a bag that exists on disk.

Getting Started

Java Web Start

The Bagger application can be initiated using Java Web Start from a Web server using a Web browser that runs Java Runtime Environment 1.6+. Refer to the release notes for your environment to determine the Web site for accessing Bagger. This will start the signed executable bagger.jar file.

Windows Stand-Alone

The file bagger.jar is a signed executable application. To start the Bagger application find the directory bagger_stndalone_win. In this directory you can double-click on the bagger_stndalone.bat file or you may:

Select Start->Run

Select the Browse button

Select the location of bagger_stndalone.bat then select the Open button

Select the OK button

The Bagger application starts with a splash banner page.

Display

The application displays the following information for the bag:
  • * The payload contents (i.e., the list of files in the data directory)
  • * The tag contents (i.e., the list of files in the base directory)
  • * The BagInfo contents
  • * The BagInfo profile
  • * If the bag exists on disk:
    • o The path/filename
    • o The format (unserialized, serialization format)
    • o The version
  • * If the bag is complete
  • * If the bag is valid
  • * If the bag has a BagInfo profile and satisfies that profile
  • * Any completion, validation, additional verify warning or errors
    Bag Content Read-Only Display
  • * Console of bag activity display tab
  • * Manifest file contents tab
  • * TagManifest file contents tab
  • * Bag-Info file contents tab
  • * BagIt file contents tab
  • * Data directory contents tab
  • * Fetch file contents tab

As the user manipulates the bag contents the results of these actions are displayed in a series of tabs on the lower left-hand side of the Bagger Application. The first tab displayed is a console that displays the state and validity of the bag as well as any messages that result during bag creation. In addition, there is a tab representing the contents of each file the bag contains according to the BagIt specification for those files that are recognized. The Data tab does not show the contents of a file, rather it displays the list of files and directories that will exist within the data directory of the bag itself.

    Console Activity Display Tab
  • * Are all required fields set?
  • * Is bag complete?
  • * Has a bag been created?
  • * Is bag valid?
  • * Log messages window

This is the topmost tab displayed on the left-hand side of the Bagger Application. It conveys the current status of any actions initiated by the user and displays messages and progress reports regarding bag creation and validation. At the top are a series of bag validation indicators based upon what makes an acceptable bag according to the BagIt specification.

    • * Are all required fields set? yes or no, are the contents of the bag-info.txt file fields valid for the project this bag belongs to. The BagIt specification allows a bag to be created even though all fields are not valid.
    • * Is bag complete? yes or no, are all the files indicated in the manifests present and accounted for.
    • * Has a bag been created? yes or no, has the bag directory been created on the local file system or has a serialized zip/tar archive file been created successfully.
    • * Is bag valid? yes or no, do all of the files indicated have valid checksum results and is the bag successfully verified according to the BagIt specification.

Below these status indicators is a log message display window. Here are shown the results of the status checks listed above and any problems or missing conditions that occurred.

  • * Add Data : default disabled
    • o Enabled after Start New Bag selected
    • o Enabled after Open Existing Bag
    • o Disables Validate Bag button
    • o Enables Save Bag As button
  • * Save Bag : default disabled
    • o Enabled after Open Existing Bag selected
    • o Enabled after Save Bag As selected
    • o Enables Validate Bag button
  • * Save Bag As : default disabled
    • o Enabled after Open Existing Bag selected
    • o Enabled after Add Data selected
    • o Enables Validate Bag button
    • o Enables Save Bag button

These buttons are used to create new bags or view existing bags. They also allow the user to add content to a bag and save the bag. Once a bag exists it can be validated. The results of bag creation and validation are displayed in the console tab. Adjacent to the buttons is a selectable checkbox tree that displays a hierarchical representation of the files and directories currently added to the bag. Selecting and deselecting a file will add or remove it to or from the bag. Selecting and deselecting a directory will add or remove the directory and all its child contents to the bag.
